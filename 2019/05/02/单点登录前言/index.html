单点登录

  • 单点登录
    single sing on(简称SSO)
    多系统，单一位置登录，实现多系统同时登陆的一种技术
    常出现在互联网应用和企业级平台中
    如：京东

技术实现

1、登录机制图：

正常用户输入密码和用户后，用户session状态变为yes，同时在浏览器中写入cookie，Cookie为用户的唯一标识。
下次登录，请求直接带cookie，通过Cookie找到对应的Session，不做特殊配置的话，Cookie名为jessionId，Cookie在服务端唯一。
注：所以平常浏览器清除Cookie之后，很多账号需要重新登录。
2、实现单点登录的问题：
Cookie不能跨域 Session不能共享

当前解决方案

同域情况下
例子：（一般情况下，一个企业只有一个域名）
  准备两个应用：app1.a.com， app2.a.com
  一个单点登录：sso.a.com
如图：

解决方法：

1、Cookie问题:
SSO登录，将Cookie的域设为顶域，即a.com，这样所有子域的系统都可以访问到顶域的Cookie
注：设置Cookie，只能设置自己的域和顶域，不能设置其他域

2、Session问题：
把三个系统的Session共享，例如Spring-Session

不同域情况下


注：第一次登录成功，更改seesion状态之后，返回一个ST，用于验证，这是为了防止直接在浏览器中敲入回调地址，并附带伪造用户信息，防止注入攻击。

总结：

  • 单点登录（SSO系统）保障各业务系统用户资源的安全。
  • 各个业务系统获得的信息为：用户有无权限访问资源
  • 单点登录，资源在各个业务系统那边，不在SSO这边，用户给SSO提供了用户名和密码之后，业务系统并不知道此事。因此SSO如果随便给业务系统一个ST，业务系统不能确定这个ST是否伪造，是否有效，所以需要重新访问SSO，以确认是否有效。
